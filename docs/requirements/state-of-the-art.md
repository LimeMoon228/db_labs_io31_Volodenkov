# Аналіз предметної області
## Вступ

У даному розділі представлено всебічний аналіз предметної області управління відкритими даними. Відкриті дані є важливим інструментом для забезпечення прозорості, підвищення ефективності управління та стимулювання інновацій. Вони надають можливість вільного доступу до інформації, що може бути використана для різних досліджень, розробки додатків та прийняття рішень.
Розуміння ключових аспектів відкритих даних, таких як стандарти форматів, механізми публікації, ліцензування та інтеграція з іншими системами, є критично важливим для створення ефективної системи управління. Ця частина документації аналізує основні терміни, підходи до побудови таких систем, а також проводить порівняльний аналіз існуючих платформ за допомогою моделі FURPS, що охоплює функціональні вимоги, зручність використання, надійність, продуктивність і підтримку

## Основні визначення
**[Відкриті дані](https://uk.wikipedia.org/wiki/%D0%92%D1%96%D0%B4%D0%BA%D1%80%D0%B8%D1%82%D1%96_%D0%B4%D0%B0%D0%BD%D1%96)** (opens new window) — це концепція, за якою певні дані мають бути вільними для використання та розповсюдження будь-якою особою з будь-якою метою.

**[Система управління базами даних](https://uk.wikipedia.org/wiki/%D0%A1%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0_%D0%BA%D0%B5%D1%80%D1%83%D0%B2%D0%B0%D0%BD%D0%BD%D1%8F_%D0%B1%D0%B0%D0%B7%D0%B0%D0%BC%D0%B8_%D0%B4%D0%B0%D0%BD%D0%B8%D1%85)** (opens new window)(СУБД, СКБД англ. Database Management System, DBMS) — набір взаємопов'язаних даних (база даних) і програм для доступу до цих даних. Надає можливості створення, збереження, оновлення та пошуку інформації в базах даних з контролем доступу до даних.

**[Мова структурованих запитів](https://uk.wikipedia.org/wiki/SQL)** (opens new window)(скор. “SQL” від англ. “Structured Query Language”)— це стандартна мова запитів, яка використовується для роботи з реляційними базами даних.

**[Інтерфейс програмування застосунків (прикладний програмний інтерфейс, API)](https://uk.wikipedia.org/wiki/%D0%9F%D1%80%D0%B8%D0%BA%D0%BB%D0%B0%D0%B4%D0%BD%D0%B8%D0%B9_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BD%D0%B8%D0%B9_%D1%96%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81)** (opens new window)— це спосіб взаємодії комп'ютерних програм між собою. Набір визначень. підпрограм, протоколів взаємодії та засобів для створення програмного забезпечення.

**[Формати даних](https://uk.wikipedia.org/wiki/%D0%A4%D0%BE%D1%80%D0%BC%D0%B0%D1%82%D0%B8_%D0%B4%D0%B0%D0%BD%D0%B8%D1%85)** — сукупність регламентованих в архітектурі ЕОМ структур для представлення різних за специфікою обробки та представлення видів інформації (цілих чисел, дробових чисел, текстових рядків тощо)

**[Теорема CAP](https://uk.wikipedia.org/wiki/%D0%A2%D0%B5%D0%BE%D1%80%D0%B5%D0%BC%D0%B0_CAP)** (opens new window)(також відома як теорема Брюера, на честь науковця Еріка Брюера) — твердження, що для будь-якої розподіленої комп'ютерної системи неможливо одночасно забезпечити виконання більше двох із перелічених трьох властивостей:
- узгодженість даних (усі вузли бачать однакові дані у будь-який момент часу);
- доступність (гарантія того, що кожен запит отримає коректну відповідь);
- стійкість до розділення (попри розділення на ізольовані секції або втрати зв'язку з частиною вузлів, система не втрачає стабільність і здатність коректно відповідати на запити).

## Підходи та способи вирішення завдання

### Методи збору даних
#### 1. [Web scraping](https://uk.wikipedia.org/wiki/Web_scraping)
Web scraping — це технологія автоматичного вилучення великих обсягів даних із вебресурсів. Цей метод дозволяє систематично сканувати сайти в інтернеті та збирати інформацію за заданими критеріями.

**Для чого потрібен Web scraping:**
- Аналіз ринку: вивчення конкурентів, їхніх пропозицій і цін, що корисно для розробки стратегії запуску системи.
- Моніторинг новин: відстеження актуальних подій чи змін у сфері відкритих даних.
- Оцінка контенту: аналіз ефективності публікацій у соціальних мережах чи блогах для визначення популярних тем.
- Підготовка даних для машинного навчання: забезпечення нейронних мереж інформацією для їхнього тренування.

**Переваги:**
- Легкий доступ до відкритої інформації.
- Автоматизація процесу збору.
- Можливість охопити широкий спектр даних.

**Недоліки:**
- Зміни в структурі сайтів можуть порушити процес збору.
- Обмеження, пов’язані з юридичними аспектами (наприклад, авторське право).
- Високі вимоги до обчислювальних ресурсів.

#### [API](https://uk.wikipedia.org/wiki/%D0%9F%D1%80%D0%B8%D0%BA%D0%BB%D0%B0%D0%B4%D0%BD%D0%B8%D0%B9_%D0%BF%D1%80%D0%BE%D0%B3%D1%80%D0%B0%D0%BC%D0%BD%D0%B8%D0%B9_%D1%96%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81)
API (інтерфейс програмування додатків) — це інструмент, який дозволяє різним програмам обмінюватися даними та взаємодіяти між собою, спрощуючи доступ до інформації.

**Типи API:**
- Відкриті (публічні).
- Партнерські.
- Внутрішні (приватні).
- Спеціалізовані (складні).

**Переваги:**
- Економія ресурсів: використання готового коду скорочує час розробки.
- Прогнозованість: API працює стабільно та зрозуміло в різних системах.

**Недоліки:**
- Обмежена глибина: розробник залежить від логіки чужого коду, що може ускладнити усунення помилок.
- Неможливість розширення: функціонал API фіксований і не завжди піддається модифікації.

#### [Витяг із бази даних (Database Extraction)](https://en.wikipedia.org/wiki/Data_extraction)
Цей метод передбачає отримання даних із бази даних за допомогою мов запитів, таких як SQL. Він дозволяє швидко й точно вилучати структуровану інформацію для подальшого використання в системі управління відкритими даними.

**Особливості:**
- Прямий доступ до структурованих даних: SQL забезпечує точне вилучення потрібної інформації.
- Висока ефективність: ідеально підходить для роботи з великими наборами даних.
- Гнучкість: можливість налаштувати запити для отримання даних за конкретними умовами.
- Маніпуляція даними: крім вилучення, SQL дозволяє оновлювати чи видаляти записи.
- Інтеграція: сумісність із програмними мовами (наприклад, Python) і аналітичними інструментами.

### Зберігання даних
#### [NoSQL бази даних](https://uk.wikipedia.org/wiki/NoSQL)
NoSQL — це підхід до зберігання даних, який відходить від традиційних реляційних моделей і використовує альтернативні структури, такі як ключ-значення, документи чи графи.

**Переваги:**
- Простота створення схеми даних.
- Легке масштабування на розподілених системах.
- Швидше виконання певних операцій порівняно з реляційними базами.

**Недоліки:**
- Можливі компроміси в узгодженості даних заради доступності.
- Використання низькорівневих запитів замість стандартизованого SQL.
- Ризик втрати даних у деяких системах (без належного журналювання).

#### [Реляційні бази даних (RDBMS)](https://uk.wikipedia.org/wiki/%D0%A0%D0%B5%D0%BB%D1%8F%D1%86%D1%96%D0%B9%D0%BD%D0%B0_%D0%B1%D0%B0%D0%B7%D0%B0_%D0%B4%D0%B0%D0%BD%D0%B8%D1%85)
Реляційні бази даних зберігають інформацію в таблицях, пов’язаних між собою через ключі, що забезпечує структурованість і цілісність даних.

**Переваги:**
- Забезпечення точності та узгодженості даних завдяки зв’язкам і обмеженням.
- Можливість масштабування разом із потребами системи.
- Підтримка складних запитів для аналізу відкритих даних.

**Недоліки:**
- Складність горизонтального масштабування порівняно з NoSQL.
- Потреба в ретельному плануванні структури бази.

#### [Гібридні бази даних](https://uk.wikipedia.org/wiki/%D0%93%D1%96%D0%B1%D1%80%D0%B8%D0%B4%D0%BD%D0%B0_%D1%96%D0%BD%D1%82%D0%B5%D0%BB%D0%B5%D0%BA%D1%82%D1%83%D0%B0%D0%BB%D1%8C%D0%BD%D0%B0_%D1%81%D0%B8%D1%81%D1%82%D0%B5%D0%BC%D0%B0)
Гібридні системи поєднують реляційні та нереляційні підходи, дозволяючи працювати з різноманітними типами даних (структурованими й неструктурованими).

**Переваги:**
- Універсальність: підтримка різних форматів даних.
- Висока продуктивність і масштабованість.

**Недоліки:**
- Складність адміністрування через комбінацію моделей.
- Вищі витрати на підтримку.

#### [Хмарні системи зберігання (Cloud Storage)](https://uk.wikipedia.org/wiki/%D0%A5%D0%BC%D0%B0%D1%80%D0%BD%D1%96_%D1%81%D1%85%D0%BE%D0%B2%D0%B8%D1%89%D0%B0)
Хмарні рішення надають віддалене зберігання даних із доступом через інтернет, що ідеально для відкритих даних із глобальним охопленням.

**Переваги:**
- Масштабованість: адаптація до обсягу даних.
- Доступність із будь-якої точки світу.
- Безпека: шифрування та резервне копіювання.

**Недоліки:**
- Залежність від стабільного інтернет-з’єднання.
- Обмеження в управлінні та відповідності правовим нормам.

### Аналіз даних
#### 1. [TensorFlow](https://www.tensorflow.org/)
TensorFlow — це відкрита бібліотека машинного навчання, розроблена Google, яка дозволяє обробляти великі обсяги даних, будувати моделі для аналізу та прогнозування, а також виявляти закономірності в інформації. У контексті системи управління відкритими даними TensorFlow може бути використано для класифікації даних, виявлення трендів чи аномалій, а також для підготовки даних до подальшого використання чи візуалізації.

**Основні можливості:**
- Моделювання та тренування: TensorFlow підтримує створення нейронних мереж, які можна навчити на відкритих даних для виконання таких завдань, як класифікація документів, прогнозування попиту на певні набори даних чи оцінка їхньої якості.
- Попередня обробка: Завдяки бібліотеці TensorFlow Transform (TFT) можна виконувати попередню обробку даних, наприклад, нормалізацію чи очищення, що є важливим для роботи з різнорідними відкритими джерелами.
- Перевірка якості: TensorFlow Data Validation (TFDV) дозволяє аналізувати статистичні характеристики даних (наприклад, пропущені значення, розподіл даних), що допомагає виявляти помилки чи невідповідності у відкритих наборах.
- Оцінка моделей: TensorFlow Model Analysis (TFMA) забезпечує оцінку ефективності моделей на великих обсягах даних у розподіленому режимі, що корисно для перевірки точності прогнозів чи класифікації.
- Управління метаданими: ML Metadata (MLMD) записує та зберігає метадані про процеси аналізу, що забезпечує відтворюваність і контроль за робочими процесами.

**Переваги:**
- Гнучкість у створенні складних моделей для аналізу відкритих даних.
- Інтеграція з Python, що полегшує використання в екосистемі обробки даних.
- Підтримка TFX (TensorFlow Extended) для автоматизації повного циклу аналізу — від збору до оцінки результатів.

**Недоліки:**
- Високі вимоги до обчислювальних ресурсів, особливо при роботі з глибокими нейронними мережами.
- Складність налаштування для початківців, оскільки потрібні знання машинного навчання.
- Орієнтація на задачі машинного навчання, що може бути надмірним для простих аналітичних операцій.

**Застосування в системі:**
TensorFlow може бути використано для автоматичного аналізу якості відкритих даних (наприклад, виявлення помилок у наборах), прогнозування їхньої популярності серед користувачів або класифікації за темами чи типами (економіка, екологія тощо).

#### 2. [Apache Spark](https://spark.apache.org/)
Apache Spark — це потужний фреймворк із відкритим кодом для обробки великих даних у розподіленому середовищі. Він ідеально підходить для аналізу відкритих даних завдяки своїй здатності швидко обробляти величезні обсяги інформації, включаючи структуровані, напівструктуровані та неструктуровані формати. Spark пропонує вбудовані бібліотеки для SQL-запитів, машинного навчання (MLlib), потокової обробки та графового аналізу.

**Основні можливості:**
- Розподілена обробка: Spark використовує кластерну архітектуру, що дозволяє паралельно обробляти дані на кількох вузлах, значно прискорюючи аналіз великих наборів відкритих даних.
- Spark SQL: Дозволяє виконувати SQL-запити до даних у різних форматах (CSV, JSON, Parquet), що спрощує роботу з різноманітними джерелами відкритих даних.
- Машинне навчання (MLlib): Надає інструменти для класифікації, регресії, кластеризації та рекомендаційних систем, які можна застосувати для аналізу трендів чи сегментації даних.
- Потокова обробка (Spark Streaming): Забезпечує аналіз даних у реальному часі, що корисно для моніторингу оновлень у відкритих джерелах.
- Інтеграція: Spark легко інтегрується з Hadoop, хмарними сховищами (наприклад, AWS S3) і базами даних, що робить його універсальним для роботи з відкритими даними.

**Переваги:**
- Висока швидкість обробки завдяки роботі в оперативній пам’яті (in-memory computing).
- Простота використання для базового аналізу через підтримку SQL і API для Python (PySpark).
- Масштабованість: здатність обробляти як невеликі, так і величезні обсяги даних без значних змін у коді.

**Недоліки:**
- Потреба в налаштуванні кластера для розподіленої роботи, що може бути складним і дорогим.
- Менша гнучкість у порівнянні з TensorFlow для глибокого машинного навчання.
- Високе споживання пам’яті при обробці великих наборів даних.

**Застосування в системі:**
Apache Spark може бути використано для агрегації та очищення відкритих даних із різних джерел, побудови аналітичних звітів (наприклад, статистики використання даних користувачами) або реального часу моніторингу змін у наборах даних (наприклад, оновлення статистики чи нових публікацій).

### Візуалізація даних
#### [Візуалізація даних із використанням D3.js](https://uk.wikipedia.org/wiki/D3.js)

D3.js (Data-Driven Documents) — це бібліотека JavaScript із відкритим кодом, призначена для створення динамічних і гнучких візуалізацій даних у веббраузерах. Вона працює на основі вебстандартів, таких як SVG (масштабована векторна графіка), HTML5 і CSS, дозволяючи розробникам створювати висококastomізовані графіки, які керуються даними. У контексті відкритих даних D3.js ідеально підходить для створення інтерактивних діаграм, карт чи графіків, які користувачі можуть досліджувати.

**Основні можливості:**
- Низькорівневий контроль: D3.js не пропонує готових шаблонів діаграм, а надає інструменти для створення візуальних елементів із нуля, що дозволяє точно налаштувати кожен аспект графіки.
- Інтерактивність: Підтримує додавання анімацій, спливаючих підказок, фільтрів і реакцій на дії користувача (наприклад, кліки чи наведення).
- Гнучкість форматів: Інтеграція з SVG і Canvas забезпечує створення як векторних, так і растрових візуалізацій, що підходить для відображення складних наборів даних (наприклад, географічних карт).
- Сумісність із вебтехнологіями: Легко інтегрується з іншими вебінструментами та бібліотеками, дозволяючи вбудовувати візуалізації в системи управління даними.

**Переваги:**
- Повний контроль над дизайном і поведінкою візуалізацій, що дозволяє створювати унікальні рішення для відкритих даних.
- Підтримка вебстандартів, що забезпечує кросбраузерну сумісність і легкість стилізації через CSS.
- Можливість створювати складні інтерактивні графіки, які ідеально підходять для дослідницького аналізу даних.

**Недоліки:**
- Висока складність розробки: створення навіть базової діаграми вимагає значного обсягу коду та знань JavaScript.
- Час затратний процес: відсутність готових шаблонів ускладнює швидке розгортання візуалізацій.
- Потреба в технічних навичках: для ефективного використання потрібен досвід веброзробки.

**Застосування в системі:**
D3.js можна використати для створення інтерактивних карт із геоданими (наприклад, розподіл відкритих даних за регіонами), складних графіків зв’язків між наборами даних або динамічних діаграм, які користувачі можуть налаштовувати в реальному часі (наприклад, фільтрувати дані за роками чи категоріями).

#### 2. [Візуалізація даних із використанням Power BI](https://uk.wikipedia.org/wiki/Power_BI)

Power BI — це потужний інструмент від Microsoft для бізнес-аналітики та візуалізації даних, який дозволяє швидко створювати інтерактивні інформаційні панелі (dashboards) і звіти. Він орієнтований на користувачів із різним рівнем технічної підготовки та підтримує інтеграцію з численними джерелами даних, включаючи бази даних, хмарні сервіси та файли. У системі управління відкритими даними Power BI може слугувати для надання доступних і зрозумілих візуалізацій широкій аудиторії.

**Основні можливості:**
- Готові шаблони: Power BI пропонує широкий вибір вбудованих типів візуалізацій (стовпчикові діаграми, графіки, карти, таблиці), які легко налаштовуються через drag-and-drop інтерфейс.
- Інтерактивність: Користувачі можуть фільтрувати, деталізувати (drill-down) і досліджувати дані безпосередньо в інтерфейсі без необхідності писати код.
- Інтеграція з даними: Підтримує підключення до SQL-баз, NoSQL, Excel, API і хмарних сховищ (наприклад, Azure), що ідеально для роботи з різноманітними джерелами відкритих даних.
- Хмарний доступ: Звіти та панелі можна публікувати онлайн, надаючи доступ користувачам через веббраузер або мобільні додатки.
- Розширені аналітичні функції: Вбудована підтримка мови DAX (Data Analysis Expressions) для створення складних обчислень і Power Query для трансформації даних.

**Переваги:**
- Простота використання: навіть користувачі без програмування можуть створювати професійні візуалізації.
- Швидкість розгортання: готові інструменти дозволяють створювати звіти за лічені хвилини.
- Широка доступність: хмарна інфраструктура та інтеграція з Microsoft-екосистемою роблять Power BI зручним для командної роботи.

**Недоліки:**
- Обмежена гнучкість: порівняно з D3.js, Power BI менш адаптивний для створення унікальних чи нестандартних візуалізацій.
- Залежність від платформи: повна функціональність доступна лише в платній версії, а безкоштовна має обмеження.
- Менша кастомізація: складно реалізувати специфічні дизайнерські рішення чи складну інтерактивність.

**Застосування в системі:**
Power BI може бути використано для створення інформаційних панелей, які відображають статистику використання відкритих даних (наприклад, кількість завантажень за категоріями), географічні карти з розподілом даних або звіти про тренди (наприклад, зростання обсягу даних за місяць). Цей інструмент підходить для надання зрозумілого інтерфейсу кінцевим користувачам, таким як аналітики чи державні службовці.

#### Порівняння та рекомендації
D3.js найкраще підходить для розробки складних, кастомізованих і високодинамічних візуалізацій, які потребують глибокої інтерактивності та унікального дизайну. Воно ідеальне для технічних користувачів або систем, де потрібен повний контроль над відображенням даних.

Power BI є оптимальним вибором для швидкого створення доступних і зрозумілих візуалізацій, які не потребують складного програмування. Воно підходить для широкої аудиторії, включаючи непрофесійних користувачів, і для стандартних аналітичних завдань.

Для системи управління відкритими даними можна комбінувати ці підходи: D3.js — для створення спеціалізованих інтерактивних візуалізацій (наприклад, дослідницьких інструментів для експертів), а Power BI — для надання простих і зручних звітів широкій аудиторії (наприклад, громадськості чи менеджерам). Такий підхід забезпечить баланс між гнучкістю та доступністю.

### Доступ до даних
#### 1. [Доступ до даних із використанням ABAC (Attribute-Based Access Control)](https://en.wikipedia.org/wiki/Attribute-based_access_control)

ABAC (контроль доступу на основі атрибутів) — це гнучка модель авторизації, яка визначає права доступу на основі характеристик (атрибутів) суб’єкта, ресурсу, дії та середовища. У системі управління відкритими даними ABAC дозволяє точно налаштувати доступ до наборів даних, враховуючи контекст запиту, що робить її ідеальною для роботи з різноманітними користувачами та типами даних.

**Основні компоненти:**
- Суб’єкт: Користувач або система, що запитує доступ. Атрибути включають ідентифікатор, групу, роль, рівень доступу чи організаційну приналежність. Наприклад, "користувач із департаменту статистики".
- Ресурс: Дані, до яких потрібен доступ (наприклад, набір відкритих даних про бюджет). Атрибути — тип даних, дата створення, рівень конфіденційності.
- Дія: Операція, яку суб’єкт хоче виконати (читання, редагування, завантаження). Наприклад, "завантажити CSV-файл".
- Середовище: Контекст запиту, як-от час доступу, IP-адреса, пристрій чи рівень шифрування. Наприклад, "доступ із корпоративної мережі в робочий час".

**Механізм роботи:**
ABAC аналізує комбінацію атрибутів і застосовує правила політики безпеки. Наприклад, правило може дозволяти доступ до фінансових даних лише користувачам із атрибутом "аналітик" із корпоративного пристрою в робочий час.

**Переваги:**
- Висока гнучкість: доступ можна налаштувати на основі будь-яких атрибутів, що ідеально для систем із різноманітними користувачами (громадськість, дослідники, держслужбовці).
- Контекстна адаптивність: враховує зовнішні фактори, такі як час чи місце, підвищуючи безпеку.
- Точність: дозволяє створювати деталізовані політики для захисту чутливих відкритих даних.

**Недоліки:**
- Складність впровадження: потребує ретельного визначення атрибутів і правил, що може бути трудомістким.
- Високі вимоги до обчислень: перевірка багатьох атрибутів у реальному часі може сповільнити систему.
- Необхідність управління метаданими: підтримка актуальності атрибутів суб’єктів і ресурсів потребує додаткових зусиль.

**Застосування в системі:**
ABAC може бути використано для надання доступу до відкритих даних із різними рівнями чутливості. Наприклад, загальнодоступні набори даних доступні всім, тоді як дані з обмеженим доступом (наприклад, статистика з грифом "для службового користування") — лише користувачам із певними атрибутами (посада, організація).

#### 2. [Доступ до даних із використанням RBAC (Role-Based Access Control)](https://uk.wikipedia.org/wiki/%D0%9A%D0%B5%D1%80%D1%83%D0%B2%D0%B0%D0%BD%D0%BD%D1%8F_%D0%B4%D0%BE%D1%81%D1%82%D1%83%D0%BF%D0%BE%D0%BC_%D0%BD%D0%B0_%D0%BE%D1%81%D0%BD%D0%BE%D0%B2%D1%96_%D1%80%D0%BE%D0%BB%D0%B5%D0%B9)

RBAC (контроль доступу на основі ролей) — це модель авторизації, яка призначає права доступу залежно від ролі користувача в системі. У контексті управління відкритими даними RBAC є простішим і широко застосовуваним підходом, який базується на чіткому розподілі обов’язків між користувачами (наприклад, адміністратор, редактор, читач).

**Основні компоненти:**
- Ролі: Визначені набори прав і обов’язків, такі як "гість", "аналітик", "адміністратор". Кожна роль має фіксований набір дозволів.
- Користувачі: Призначаються до однієї чи кількох ролей залежно від їхньої функції. Наприклад, "аналітик" може мати доступ до читання й аналізу даних.
- Дозволи: Конкретні дії, які роль може виконувати (читання, запис, видалення). Наприклад, "гість" може лише переглядати дані, а "адміністратор" — редагувати їх.

**Механізм роботи:**
Користувачу надається роль, а система автоматично застосовує відповідні дозволи. Наприклад, "редактор" може оновлювати набори даних, тоді як "гість" — лише переглядати їх у вебінтерфейсі.

**Переваги:**
- Простота впровадження: чітка структура ролей полегшує налаштування та адміністрування.
- Ефективність: менш ресурсоємна порівняно з ABAC, оскільки не потребує аналізу багатьох атрибутів.
- Масштабованість: легко додавати нових користувачів, призначаючи їм існуючі ролі.

**Недоліки:**
- Обмежена гнучкість: RBAC не враховує контекст (наприклад, час чи пристрій), що може бути критичним для відкритих даних із різними рівнями доступу.
- Статичність: зміна прав потребує оновлення ролей, що може ускладнити управління в динамічних системах.
- Ризик надмірних прав: користувач із роллю може отримати доступ до даних, які йому не потрібні.

**Застосування в системі:**
RBAC підходить для базового розмежування доступу в системі відкритих даних. Наприклад, роль "громадський користувач" дозволяє лише переглядати дані, "дослідник" — завантажувати набори для аналізу, а "адміністратор" — додавати чи видаляти дані. Цей підхід зручний для систем із чітко визначеними групами користувачів.

#### Порівняння та рекомендації
ABAC є кращим вибором для складних систем із різноманітними сценаріями доступу, де потрібна висока точність і контекстна залежність. Наприклад, для відкритих даних із різними рівнями конфіденційності чи доступом, що залежить від часу чи місця.

RBAC підходить для простіших систем із фіксованими ролями та меншою потребою в динамічному управлінні доступом. Воно ідеальне для початкового етапу впровадження чи систем із обмеженою кількістю типів користувачів.

Для системи управління відкритими даними можна комбінувати ці підходи: RBAC — для базового розподілу прав між основними групами користувачів (наприклад, гості, редактори), а ABAC — для тонкого налаштування доступу до специфічних наборів даних (наприклад, обмеження доступу до чутливих даних за контекстом). Такий гібридний підхід забезпечить баланс між простотою й безпекою.

## Висновки

У сучасному цифровому середовищі управління відкритими даними є критично важливим, але наявні платформи, такі як CKAN чи OpenDataSoft, обмежені у функціональності та адаптивності. Розробка нової інформаційної системи є доцільною, оскільки вона може об’єднати ефективний збір, аналіз і представлення даних, перевершуючи існуючі рішення. Для забезпечення універсальності необхідна інтеграція з зовнішніми сервісами, наприклад, Google Cloud чи API урядових порталів. Хоча модифікація наявних систем можлива, нова розробка дозволить повніше відповідати поточним потребам. Поєднання з інструментами типу TensorFlow для аналізу та D3.js для візуалізації підвищить її конкурентоспроможність. Отже, створення нової системи є обґрунтованим і перспективним кроком.
